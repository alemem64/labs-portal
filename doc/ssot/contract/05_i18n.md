# Contract 05 — 다국어 (ko / en)

## 원칙

- 지원 locale은 `"ko" | "en"` 정확히 2개 (MUST). `Locale` 타입의 정의 위치는 `src/i18n/locales.ts`이며 다른 파일은 여기서 import한다 (MUST).
- i18n 라이브러리 금지 (01_stack.md). React Context 기반 자체 구현.

## 문자열 관할

| 종류 | 위치 |
|---|---|
| 카드 title/desc | `src/data/services.ts`의 `LocalizedText` 필드 |
| UI 공통 문자열 (버튼 라벨 등) | `src/i18n/locales.ts`의 `messages` 테이블 |

- 컴포넌트에 한국어/영어 문자열 리터럴 하드코딩 금지 (MUST). 반드시 `t(key)` 또는 `title[locale]` 경유.
- `messages`는 `satisfies Record<Locale, ...>`로 두 언어 키 누락을 타입 차단한다 (MUST).
- 예외: `index.html`의 메타 태그·noscript·부트 로고는 정적이므로 두 언어 병기 또는 브랜드 고유명사만 사용.

## Locale 결정 순서 (MUST)

1. `localStorage["lsl.locale"]` 값이 `"ko"` 또는 `"en"`이면 그것.
2. 아니면 `navigator.language`가 `ko`로 시작하면 `"ko"`, 그 외 `"en"`.
3. localStorage 접근 실패(프라이빗 모드 등)는 조용히 무시하고 2번으로.

## 전환 UI

- 화면 우상단 고정 토글 버튼 1개. 클릭 시 즉시 전환 + `localStorage["lsl.locale"]`에 저장 (MUST).
- 버튼 라벨은 **전환될 대상 언어**를 표기한다 (ko 상태 → "EN", en 상태 → "한국어").
- locale 변경 시 `document.documentElement.lang`을 갱신한다 (MUST).
- URL로 locale을 전달하지 않는다 — 해시는 card_id 전용(04_interaction.md).
